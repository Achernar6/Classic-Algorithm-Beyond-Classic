# Markdown File Index

This index maps the project's Markdown files to their current responsibilities.

## Root

- `DEVELOPMENT_ROADMAP.md`: MVP issue specifications, build order, definition of done, and pointers to supporting docs.
- `README.md`: public-facing project overview and contributor entry point.
- `[ign]AI_CODING_PROMPTS.md`: ignored working prompt notes.
- `[ign]HumanNote.md`: ignored human notes.

## Docs

- `docs/project_philosophy.md`: project purpose, teaching philosophy, notebook-first rule, and scientific guardrails.
- `docs/math_overview.md`: mathematical source of truth for layered DAG paths, DP, Soft-DP, QA, observables, and invariants.
- `docs/qa_proxy_notes.md`: detailed boundary notes for the feasible-subspace QA proxy.
- `docs/visualization_guide.md`: intended visual quantities and plotting semantics.
- `docs/notebook_acceptance_guide.md`: notebook-visible acceptance criteria by issue.
- `docs/future_roadmap.md`: post-MVP directions kept out of the main execution roadmap.
- `docs/execution_log.md`: completed issue execution notes moved out of the main roadmap.
- `docs/markdown_file_index.md`: this Markdown file directory.
